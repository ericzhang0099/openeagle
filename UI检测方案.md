# AI Agent 前端 UI 检测方案

## 一、检测方法论

### 1. 静态代码分析
- 读取 HTML/JS 文件
- 搜索 onclick, addEventListener
- 检测函数是否定义
- 正则匹配语法错误

### 2. 浏览器运行时检测 (推荐)
使用 Playwright / Puppeteer：
- 自动打开页面
- 模拟用户点击
- 捕获 Console 错误
- 检测 JS 异常
- 截图记录

### 3. DOM 结构检测
- 检查元素是否存在
- 验证表单 action
- 检测死链

---

## 二、检测清单

### 2.1 登录功能检测
- [ ] 登录按钮 onclick 调用的函数是否存在
- [ ] 函数内部是否有 API 调用
- [ ] API 路径是否正确
- [ ] 错误处理是否完善

### 2.2 导航功能检测
- [ ] 所有导航按钮的 onclick 是否定义
- [ ] showPage() 函数是否存在
- [ ] renderPage() 函数是否存在
- [ ] 页面切换是否正常工作

### 2.3 表单检测
- [ ] input 元素是否有 id/name
- [ ] 表单提交是否阻止默认行为
- [ ] 验证逻辑是否存在

### 2.4 运行时错误检测
- [ ] JS 语法错误
- [ ] 未定义变量/函数
- [ ] API 请求失败
- [ ] CORS 错误

---

## 三、Agent 自动化检测流程

```python
# 伪代码
1. 读取所有 HTML 文件
2. 提取所有 onclick 属性值
3. 提取所有 addEventListener 事件
4. 在文件中搜索这些函数名
5. 标记未定义的函数
6. 使用浏览器打开页面
7. 模拟点击按钮
8. 捕获 console.error
9. 输出检测报告
```

## 四、常见前端问题

| 问题类型 | 检测方法 | 典型症状 |
|----------|----------|----------|
| 函数未定义 | 静态分析 | 点击无反应 |
| API 路径错误 | 运行时检测 | Network 报错 |
| CORS 问题 | 浏览器检测 | Console 报错 |
| CSS 加载失败 | 浏览器检测 | 页面显示异常 |
| JS 语法错误 | 浏览器检测 | Console 红色错误 |

---

## 五、工具推荐

1. **Playwright** - 浏览器自动化测试
2. **Puppeteer** - Chrome 自动化
3. **Lighthouse** - 前端性能和安全
4. **ESLint** - JS 语法检查
5. **Puppeteer-recorder** - 录制用户操作
