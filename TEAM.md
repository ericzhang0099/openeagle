## 二、质量保证（不降低）

### 代码质量标准

| 标准 | 要求 |
|------|------|
| **单元测试** | 核心模块覆盖率 >80% |
| **代码Review** | 每日Review，未通过不合并 |
| **类型标注** | 全部使用类型提示 |
| **文档注释** | 关键函数有docstring |
| **错误处理** | 完整的异常捕获和日志 |
| **命名规范** | 遵循PEP8 |

### Review流程

```
代码提交
    │
    ▼
CI检查 (语法/类型/测试)
    │
    ▼
技术负责人Review
    │
    ├── 通过 → 合并
    │
    └── 拒绝 → 修改后重新提交
```

## 六、沟通机制

### 每日流程

```
9:00  晨会 (15分钟)
      - 昨日完成

      - 阻塞问题

12:00 代码同步 (5分钟)
      - Git commit

18:00 进度同步 (10分钟)
      - 当日总结

19:00 Code Review (30分钟)
      - 技术负责人Review
```

### 沟通渠道
